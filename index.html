HTML5와 시맨틱 태그

시멘틱 HTML 참조 블로그

HTML5는 웹 표준을 최신화하고, 웹 애플리케이션을 더 효율적으로 만들기 위해 여러 새로운 기능과 태그를 도입한 버전입니다. 시맨틱 태그는 HTML5의 핵심 기능 중 하나로, 문서의 의미와 구조를 명확히 나타내는 태그들을 제공합니다. 이러한 태그는 접근성, SEO(검색 엔진 최적화), 유지보수 측면에서 많은 이점을 제공합니다. 여기서는 HTML5 문서의 구조와 시맨틱 태그 사용법, 그리고 IE8 이하 버전에서의 대응 방법을 다루겠습니다.

1. HTML5 문서

HTML5 문서의 기본 구조는 다음과 같습니다. <!DOCTYPE html>을 사용하여 HTML5 문서임을 선언하고, HTML 요소와 메타 정보들을 포함합니다.

HTML5 문서 구조

  • <!DOCTYPE html>: HTML5 문서임을 선언하는 태그입니다. 이는 웹 브라우저가 HTML5를 올바르게 렌더링하도록 지시합니다.
  • <html lang="ko">: lang 속성은 문서의 기본 언어를 정의합니다. 여기서는 한국어(ko)를 설정했습니다.
  • <meta charset="UTF-8">: 문자 인코딩을 UTF-8로 설정하여 다양한 언어를 지원합니다.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: 반응형 웹 디자인을 지원하는 메타 태그입니다. 화면 크기에 맞게 페이지가 자동으로 조정됩니다.
  • <title>: 브라우저 탭에 표시되는 페이지 제목을 설정합니다.

2. 문서 구조를 위한 HTML5 시맨틱 태그

HTML5 시맨틱 태그는 문서의 구조를 정의하는 데 중요한 역할을 합니다. 각 시맨틱 태그는 웹 페이지의 의미 있는 부분을 나타내므로, 검색 엔진과 사용자에게 더 잘 이해될 수 있습니다. 대표적인 시맨틱 태그는 다음과 같습니다:

2.1. <header>

  • 페이지나 섹션의 헤더를 정의합니다. 보통 로고, 내비게이션, 제목 등이 포함됩니다.

2.2. <footer>

  • 페이지나 섹션의 푸터를 정의합니다. 주로 저작권, 연락처, 사이트 맵 등이 포함됩니다.

2.3. <nav>

  • 내비게이션 링크를 정의하는 태그로, 사이트의 메뉴나 내비게이션 바 등을 포함합니다.

2.4. <main>

  • 문서의 주요 콘텐츠를 감싸는 태그로, 페이지에서 중요한 콘텐츠를 나타냅니다. 한 페이지에 하나만 사용해야 합니다.

2.5. <section>

  • 문서의 독립적인 섹션을 정의합니다. 보통 제목을 포함하며, 여러 섹션으로 나누어 페이지를 구성할 수 있습니다.

2.6. <article>

  • 독립적이고 자립적인 콘텐츠를 정의하는 태그로, 블로그 게시물, 뉴스 기사 등이 이에 해당합니다.

2.7. <aside>

  • 문서의 부수적인 콘텐츠를 정의합니다. 사이드바, 관련 기사, 광고 등을 포함하는 데 사용됩니다.

2.8. <figure> 및 <figcaption>

  • 이미지나 다이어그램 등 콘텐츠에 대한 설명을 추가할 때 사용됩니다.

3. IE8 이하 버전에서는 어떻게 하나요?

HTML5 시맨틱 태그는 최신 브라우저에서 잘 지원되지만, Internet Explorer 8 이하 버전에서는 시맨틱 태그를 제대로 지원하지 않습니다. 이 경우, 시맨틱 태그들이 기본적으로 block-level 요소로 동작하지 않기 때문에, 이를 해결하기 위해 JavaScript를 사용하여 HTML5 시맨틱 태그를 인식할 수 있도록 도와줘야 합니다.

해결 방법: HTML5 Shiv

HTML5 Shiv는 IE8 이하에서 HTML5의 새로운 시맨틱 태그를 인식할 수 있도록 해주는 JavaScript 라이브러리입니다. 이 라이브러리는 HTML5 태그들이 제대로 스타일링되고 동작하도록 만들어줍니다.

  1. HTML5 Shiv 스크립트 추가: 아래와 같이 <head>에 추가하여 HTML5 시맨틱 태그를 지원할 수 있습니다.

  2. 사용법:

    • 위 스크립트를 추가하면 IE8 이하 버전에서도 HTML5 시맨틱 태그가 제대로 작동합니다. 이 방식은 모든 최신 브라우저에서 동작하며, 구형 브라우저에서는 div와 같은 기본 블록 레벨 요소로 처리됩니다.

요약

  1. HTML5 문서는 <!DOCTYPE html>로 시작하여, 기본적인 HTML5 요소(header, footer, nav, main, section, article, aside, figure)를 포함합니다.
  2. 시맨틱 태그는 웹 페이지의 구조를 명확히 하고, SEO와 접근성을 향상시키는 중요한 역할을 합니다. 각 시맨틱 태그는 콘텐츠의 의미를 나타냅니다.
  3. IE8 이하 버전에서는 HTML5 시맨틱 태그를 지원하지 않으므로, HTML5 Shiv JavaScript 라이브러리를 사용하여 이를 해결할 수 있습니다.

HTML5와 시맨틱 태그는 웹 개발에서 중요한 부분을 차지하며, 최신 브라우저에서는 강력한 기능을 제공하고, 구형 브라우저에서는 호환성 문제를 해결할 수 있도록 지원됩니다.

시맨틱 태그를 사용하는 장점

  1. 의미가 명확함: 각 태그가 어떤 역할을 하는지 명확하게 나타내므로, 페이지 구조가 보다 이해하기 쉬워집니다.
  2. 웹 표준 준수: 시맨틱 HTML은 웹 표준을 따르므로, 웹 페이지가 다양한 브라우저에서 일관되게 작동할 확률이 높습니다.
  3. SEO 최적화: 검색 엔진이 페이지의 의미를 더 잘 파악하여, 웹 페이지의 랭킹을 높이는 데 유리합니다.
  4. 접근성 향상: 스크린 리더와 같은 보조 기술이 시맨틱 태그를 사용하여 콘텐츠를 정확하게 읽고 전달할 수 있습니다.

결론

시맨틱 HTML은 웹 페이지의 의미와 구조를 명확하게 정의할 수 있는 중요한 방법입니다. 이를 사용함으로써 웹 페이지가 보다 접근 가능하고, SEO 최적화가 잘 되며, 다른 개발자와 협업할 때 가독성 또한 향상됩니다. 시맨틱 태그를 활용하는 것은 웹 개발에서 필수적인 부분으로, 웹 페이지의 품질을 높이는 중요한 요소입니다.

HTML5와 멀티미디어

HTML5는 멀티미디어 기능을 지원하는 새로운 요소들을 도입하여, 웹 페이지에서 비디오, 오디오, 애니메이션 등 다양한 미디어 콘텐츠를 쉽게 구현할 수 있도록 도와줍니다. HTML5는 특히 <audio> 와 <video> 태그를 통해 웹 페이지에서 멀티미디어 콘텐츠를 제어하고 표시할 수 있습니다. 이 외에도 동영상을 대문처럼 활용하는 방법도 포함됩니다.

1. 웹과 멀티미디어

웹과 멀티미디어는 웹 페이지에 텍스트와 이미지만을 넘어서, 오디오, 비디오, 애니메이션 등을 포함한 다양한 콘텐츠를 제공하는 데 중요한 역할을 합니다. HTML5는 이러한 멀티미디어 콘텐츠를 웹에서 직접 삽입하고 제어할 수 있는 기능을 제공합니다.

  • 웹 페이지에서 멀티미디어 콘텐츠를 추가하면 사용자에게 더 몰입감 있는 경험을 제공할 수 있습니다.
  • HTML5는 <audio> 와 <video> 태그를 통해 별도의 플러그인 없이도 오디오와 비디오 파일을 재생할 수 있도록 지원합니다.

2. 오디오 & 비디오 재생하기

HTML5는 <audio> 와 <video> 태그를 통해 오디오 및 비디오 파일을 쉽게 웹 페이지에 삽입하고 제어할 수 있습니다. 이 태그들은 다양한 속성을 제공하여 사용자가 원하는 방식으로 오디오와 비디오를 제어할 수 있게 합니다.

2.1. <audio> 태그

<audio> 태그는 오디오 파일을 웹 페이지에서 직접 재생할 수 있도록 합니다. HTML5에서는 오디오 파일을 제어할 수 있는 다양한 속성들을 제공합니다.

구문:

  • controls: 재생, 일시 정지, 볼륨 조정 등의 기본 제어 버튼을 추가합니다.
  • <source>: 오디오 파일의 경로와 형식을 지정합니다.
  • type: 오디오 파일의 MIME 타입을 지정합니다.

  • controls: 비디오의 재생, 일시 정지, 음소거, 전체화면 등의 기본 제어 버튼을 추가합니다.

3. 동영상으로 홈페이지 대문 만들기

HTML5에서는 동영상을 홈페이지의 대문(hero section)으로 활용할 수 있습니다. <video> 태그를 사용하여 배경으로 비디오를 설정하거나, 방문자가 페이지에 들어왔을 때 자동으로 재생되는 동영상을 설정할 수 있습니다.

3.1. 비디오 배경으로 설정하기

페이지의 배경으로 동영상을 설정하여 시각적인 효과를 추가할 수 있습니다. <video> 태그의 autoplay, loop, muted 속성을 사용하여 페이지 로드 시 자동으로 비디오가 재생되도록 할 수 있습니다.

구문:

  • autoplay: 페이지가 로드되자마자 비디오가 자동으로 재생됩니다.
  • loop: 비디오가 끝나면 자동으로 처음부터 반복됩니다.
  • muted: 비디오의 소리를 꺼둡니다.

3.2. CSS로 스타일링하여 대문 영역에 배경 비디오 설정

동영상을 배경 비디오로 설정할 때는 CSS로 비디오 크기와 위치를 조정할 수 있습니다. 보통 화면 크기에 맞게 비디오가 꽉 차도록 스타일링합니다.

예시 코드:

  • object-fit: cover: 비디오가 화면의 크기에 맞게 자동으로 조정되며, 비디오가 화면을 꽉 채우도록 합니다.
  • position: absolute: 콘텐츠가 비디오 위에 중앙에 위치하도록 설정합니다.

3.3. 비디오로 홈페이지 대문 만들기 팁

  • 최적화된 비디오 사용: 대용량 비디오는 페이지 로딩 속도를 느리게 만들 수 있기 때문에, 비디오 파일을 최적화하여 용량을 줄이고, 여러 포맷(mp4, webm, ogg)을 제공하는 것이 좋습니다.
  • 반응형 웹 디자인: 비디오가 화면 크기에 맞게 자동으로 크기가 조정되도록 설정합니다. (object-fit: cover 또는 background-size: cover)
  • 배경 비디오와 콘텐츠의 대비: 콘텐츠가 비디오 배경과 잘 보이도록 텍스트 색상이나 그림자 등을 추가하여 가독성을 높입니다.

요약

  1. 오디오 & 비디오 재생하기:

    • HTML5에서 <audio> 와 <video> 태그를 사용하여 오디오와 비디오 파일을 직접 웹 페이지에 삽입하고 제어할 수 있습니다.
    • controls, autoplay, loop, muted 등 다양한 속성을 통해 오디오와 비디오의 재생 방식을 조정할 수 있습니다.

  2. 동영상으로 홈페이지 대문 만들기:

    • 배경 비디오를 활용하여 시각적으로 매력적인 대문을 만들 수 있습니다.
    • object-fit: cover 와 같은 CSS 속성을 사용하여 비디오를 화면 크기에 맞게 조정하고, autoplay, loop, muted 속성을 사용하여 자동으로 재생되도록 설정할 수 있습니다.

HTML5와 멀티미디어 태그를 활용하면 웹 페이지에 더욱 풍성하고 몰입감 있는 콘텐츠를 제공할 수 있습니다.

다재다능한 CSS3 선택자

CSS3 선택자는 웹 페이지의 스타일을 적용할 HTML 요소를 선택 하는 방법을 다양하게 제공하며, 더 정교하고 강력한 스타일링을 가능하게 합니다. CSS3에서 추가된 선택자들은 특정 요소나 속성, 상태에 대한 스타일을 더 쉽게 적용할 수 있게 해줍니다. 여기서는 연결 선택자, 속성 선택자, 가상 클래스와 가상 요소 에 대해 설명합니다.

3. 가상 클래스와 가상 요소 (Pseudo-Classes & Pseudo-Elements)

CSS의 가상 클래스와 가상 요소는 특정 상태나 특정 위치에 있는 요소에 스타일을 적용할 수 있는 강력한 도구입니다.

3.1. 가상 클래스 (Pseudo-Classes)

가상 클래스는 특정 상태나 상호작용에 따라 스타일을 적용합니다.

  • :hover: 사용자가 요소에 마우스를 올렸을 때 스타일을 적용합니다.

  • :focus: 입력 필드나 링크가 포커스를 받을 때 스타일을 적용합니다.

  • :active: 요소가 활성화된 상태일 때 스타일을 적용합니다. (주로 클릭된 상태)

  • :nth-child(): 부모 요소의 자식 요소 중 특정 순서에 있는 요소를 선택합니다.

  • :first-child: 부모 요소의 첫 번째 자식 요소를 선택합니다.

  • :last-child: 부모 요소의 마지막 자식 요소를 선택합니다.

4. 인접 형제 선택자 (A + B)

A 바로 다음에 오는 형제 요소 B 를 선택

→ h1 바로 뒤에 나오는 img 에만 적용

5. 전체 구조와 함께 쓰기 좋은 예시

→ div의 자식 p의 자식인 strong 요소에 적용 (자손 관계 명확히 지정)

3.2. 가상 요소 (Pseudo-Elements)

가상 요소는 실제 존재하지 않는 요소에 스타일을 적용하는 방법입니다. 주로 텍스트나 콘텐츠의 특정 부분에 스타일을 적용할 때 사용됩니다.

  • ::before: 요소의 앞부분에 콘텐츠를 삽입합니다.

  • ::after: 요소의 뒤부분에 콘텐츠를 삽입합니다.

  • ::first-letter: 요소의 첫 번째 글자에 스타일을 적용합니다.

  • ::first-line: 요소의 첫 번째 줄에 스타일을 적용합니다.

요약

  1. 연결 선택자는 여러 HTML 요소 간의 관계에 따라 스타일을 적용하는 선택자로, 자식, 후손, 형제 요소 간의 관계를 설정할 수 있습니다. (>, 공백, ~, +)
  2. 속성 선택자는 특정 HTML 요소의 속성값에 따라 스타일을 적용합니다. 예를 들어, 속성 값이 특정 값으로 시작하거나 끝나는 요소를 선택할 수 있습니다. ([attribute="value"], [attribute^="value"], [attribute$="value"], [attribute*="value"])
  3. 가상 클래스는 요소의 특정 상태에 스타일을 적용하고, 가상 요소는 요소의 특정 부분에 스타일을 적용합니다. (:hover, :focus, ::before, ::after, ::first-letter, ::first-line)

이 선택자들을 활용하면 HTML 요소를 보다 정교하고 세밀하게 스타일링할 수 있습니다.

CSS3와 애니메이션

CSS3와 애니메이션은 웹 페이지에 동적인 효과를 추가하는 데 중요한 역할을 합니다. 변형(transform)과 트랜지션(transition), 애니메이션(animation) 등을 사용하여 사용자 경험을 향상시키고, 시각적으로 매력적인 콘텐츠를 만들 수 있습니다. 아래에서는 변형, 변형과 관련된 속성들, 트랜지션, 애니메이션에 대해 자세히 설명하고, 상품 페이지에 가격 떠오르게 만들기 예제를 제공합니다.

css 생성기

1. 변형 (Transform)

transform 속성은 요소의 모양이나 위치를 변경할 때 사용됩니다. 변형은 요소의 크기, 위치, 회전 등을 바꿀 수 있습니다. 이 속성은 2D 및 3D 변형을 지원합니다.

1.1. 주요 변형 함수

  • rotate(): 요소를 회전시킵니다. 단위는 도(degrees) 입니다.

  • scale(): 요소의 크기를 확대하거나 축소합니다.

  • translate(): 요소를 X축과 Y축 방향으로 이동시킵니다.

  • skew(): 요소를 기울입니다.

  • matrix(): 2D 변형을 한 번에 정의하는 함수로, 행렬을 사용하여 변형을 적용합니다.

2. 변형과 관련된 속성들

변형을 사용할 때 관련 속성들을 함께 사용하여 애니메이션 효과를 적용하거나 스타일을 수정할 수 있습니다. 변형을 사용할 때 자주 사용되는 속성은 다음과 같습니다:

2.1. transform-origin

  • 변형이 적용되는 기준점을 설정합니다. 기본적으로 center center로 설정되어 있지만, 이 속성으로 기준점을 변경할 수 있습니다.

2.2. perspective

  • 3D 변형 효과를 적용할 때 원근감을 설정합니다. 이 속성은 부모 요소에 적용하여 자식 요소가 3D 효과로 변형될 때 깊이를 조정합니다.

3. 트랜지션 (Transition)

트랜지션은 요소의 스타일이 변경될 때 그 변화를 부드럽게 나타내는 효과입니다. 트랜지션은 상태 변화(예: 마우스 오버, 클릭 등)에 따라 요소의 스타일을 자연스럽게 변환할 수 있게 합니다.

3.1. transition 속성

  • transition 속성은 변경될 속성, 변화 시간, 타이밍 함수, 지연 시간을 설정할 수 있습니다.

구문:

  • property: 변화를 적용할 속성입니다.
  • duration: 변화가 일어날 시간입니다. 예: 1s (1초), 500ms (500밀리초).
  • timing-function: 변화를 위한 타이밍 함수입니다. ease, linear, ease-in, ease-out, cubic-bezier 등이 있습니다.
  • delay: 변화가 시작되기 전에 기다릴 시간입니다.

  2. 트랜지션 (Transition): 요소의 스타일 변화가 부드럽게 이루어지도록 하는 속성으로, 상태 변화 시의 변화를 시간을 두고 적용합니다.
  3. 애니메이션 (Animation): @keyframes를 사용하여 요소의 스타일 변화를 지속적이고 반복적으로 적용할 수 있습니다.
  4. 상품 페이지 가격 애니메이션: CSS 애니메이션을 사용하여 상품 페이지에서 가격이 떠오르는 효과를 만들 수 있습니다.

이 기술들은 웹 페이지의 시각적 효과와 동적인 콘텐츠 구현에 매우 유용하게 활용됩니다.

반응형 웹이란?

반응형 웹 디자인(Responsive Web Design)은 다양한 화면 크기와 해상도를 가진 장치에서 웹 페이지의 레이아웃과 콘텐츠가 자동으로 최적화되도록 만드는 웹 디자인 기법입니다. 반응형 웹 디자인을 사용하면, 웹 페이지가 데스크탑, 태블릿, 모바일 등 다양한 기기에서 동일한 품질로 표시되며, 사용자 경험을 향상시킬 수 있습니다.

반응형 웹을 구현하기 위한 핵심 기술은 가변 그리드 레이아웃, 미디어 쿼리, 가변 레이아웃과 요소입니다. 아래에서 이들에 대해 설명합니다.

1. 모바일 기기와 웹 디자인

모바일 웹 디자인은 다양한 스마트폰과 태블릿 화면 크기에 맞춰 웹 페이지를 최적화하는 과정을 포함합니다. 기존에는 웹 디자인이 데스크탑 중심으로 이루어졌지만, 모바일 기기의 사용 증가로 인해 모바일 우선(mobile-first) 디자인 접근 방식이 중요해졌습니다.

모바일 기기에서 웹 디자인의 주요 요소:

  • 디스플레이 크기: 모바일 장치는 화면 크기가 작기 때문에, 모바일 웹 디자인은 화면 공간을 효율적으로 활용해야 합니다.
  • 터치스크린 인터페이스: 모바일 기기에서 터치스크린을 사용하는 사용자 경험을 고려하여 버튼 크기, 링크 간격 등을 설계합니다.
  • 빠른 로딩 속도: 모바일 데이터 네트워크를 고려해 페이지 로딩 속도가 빠르고 효율적으로 최적화되어야 합니다.
  • 자동 크기 조정: 반응형 웹 디자인을 적용하여, 화면 크기에 따라 컨텐츠가 자동으로 조정되도록 합니다.

물론입니다! 웹 개발에서 **viewport(뷰포트)**는 사용자가 보고 있는 화면 영역을 의미하며, 특히 **반응형 웹 디자인(responsive design)**에서 매우 중요한 개념입니다.

✅ Viewport란?

Viewport는 웹 페이지를 표시할 때, 브라우저나 기기의 화면에서 실제로 보여지는 영역을 의미합니다.

  • 데스크톱에서는 브라우저 창
  • 모바일에서는 전체 화면 (상태바 제외)

✅ 왜 중요할까?

모바일 기기에서는 화면 크기가 작기 때문에, 기본적으로 브라우저는 웹사이트를 축소해서 보여줍니다. 이걸 제어하려면 viewport 메타 태그를 설정해야 합니다.

2. 가변 그리드 레이아웃 (Flexible Grid Layout)

가변 그리드 레이아웃은 반응형 웹 디자인에서 가장 중요한 요소 중 하나로, 비율 기반의 레이아웃을 사용하여 다양한 화면 크기에 맞게 콘텐츠가 유동적으로 변하도록 합니다.

2.1. 그리드 시스템의 개념

  • 고정 크기 대신 비율(%) 을 사용하여 각 요소의 크기를 설정합니다. 이렇게 하면, 요소가 부모 컨테이너의 크기에 비례하여 늘어나거나 줄어듭니다.

  • fr(fractional unit)은 화면 크기에 맞춰 가변적인 크기를 설정하는 유닛입니다.

2.3. 가변 그리드와 미디어 쿼리

  • 미디어 쿼리를 사용하여 화면 크기가 다를 때 그리드 열의 개수를 변경하거나, 아이템 크기를 조정할 수 있습니다.

3. 가변 레이아웃과 가변 요소 (Flexible Layout and Flexible Elements)

반응형 웹 디자인에서는 가변 레이아웃과 가변 요소를 사용하여 콘텐츠가 화면 크기에 맞게 유연하게 조정되도록 합니다.

3.1. 가변 레이아웃 (Flexible Layout)

가변 레이아웃은 픽셀 단위 대신 비율(%), 뷰포트 단위(vw, vh), flex 와 같은 유연한 단위를 사용하여 요소를 배치합니다.

  • %: 요소의 크기를 부모 요소의 비율로 설정합니다.
  • vw (viewport width): 화면 너비의 1%에 해당하는 크기입니다.
  • vh (viewport height): 화면 높이의 1%에 해당하는 크기입니다.

3.2. flexbox를 이용한 가변 레이아웃

Flexbox는 가변적인 레이아웃을 쉽게 구현할 수 있게 해주는 CSS 레이아웃 모델입니다. Flexbox를 사용하면 요소들이 주어진 공간에 맞게 유동적으로 배치됩니다.

  • flex: 1 1 200px 는 요소가 200px로 시작하고, 부모 요소의 크기에 따라 유동적으로 크기가 변할 수 있도록 설정합니다.
  • flex-wrap: wrap 은 자식 요소들이 화면 크기에 맞게 줄바꿈 되도록 합니다.

3.3. 가변 요소 (Flexible Elements)

요소를 flexible하게 만들기 위해, flex, min-width, max-width 등의 속성을 사용하여 크기 조정이 가능하도록 합니다.

  • min-width: 요소가 최소 크기를 지정합니다.
  • max-width: 요소가 최대 크기를 지정합니다.

이 설정을 통해 <div> 요소는 최소 150px, 최대 300px의 크기로 반응형으로 크기가 조정됩니다.

결론

반응형 웹 디자인은 다양한 기기에서 최적화된 웹 경험을 제공하기 위한 핵심 기술입니다. 이를 위해 다음과 같은 기법들이 사용됩니다:

  1. 모바일 기기와 웹 디자인: 모바일-first 접근 방식으로 모바일 기기에서도 잘 작동하도록 디자인합니다.
  2. 가변 그리드 레이아웃: 비율 기반의 레이아웃을 사용하여 다양한 화면 크기에 적응할 수 있도록 설정합니다. fr 유닛을 사용한 그리드 레이아웃과 미디어 쿼리로 화면 크기에 맞춰 레이아웃을 변화시킬 수 있습니다.
  3. 가변 레이아웃과 가변 요소: flexbox 와 같은 기술을 사용하여 요소들이 화면 크기에 맞게 유동적으로 조정되도록 설정합니다.

이러한 기술들을 활용하면 모바일, 태블릿, 데스크탑 등 다양한 장치에서 일관된 사용자 경험을 제공할 수 있습니다.

미디어 쿼리 (Media Queries)

미디어 쿼리는 반응형 웹 디자인에서 중요한 역할을 하는 CSS 기능으로, 웹 페이지가 다양한 화면 크기나 기기 종류에 맞게 스타일을 조정할 수 있게 해줍니다. 이를 통해 웹 페이지는 데스크탑, 태블릿, 모바일 등에서 자동으로 최적화된 레이아웃을 제공할 수 있습니다. 미디어 쿼리는 화면 크기, 해상도, 오리엔테이션(세로/가로 모드) 등을 기반으로 특정 스타일을 적용할 수 있습니다.

1. 미디어 쿼리 이해하기

미디어 쿼리는 @media 규칙을 사용하여 특정 조건을 만족하는 화면에서만 스타일을 적용할 수 있도록 하는 CSS 기능입니다. 예를 들어, 화면의 너비가 600px 이상일 때와 600px 이하일 때 서로 다른 스타일을 적용할 수 있습니다.

1.1. 미디어 쿼리 구문

  • media-type: 미디어 유형을 정의합니다. 예: screen, print, all 등.
  • condition: 조건을 설정합니다. 예: min-width, max-width, orientation 등.

1.2. 조건 예시

  • min-width: 최소 너비 조건입니다. 이 조건은 화면이 지정된 크기 이상일 때 적용됩니다.
  • max-width: 최대 너비 조건입니다. 이 조건은 화면이 지정된 크기 이하일 때 적용됩니다.
  • orientation: 화면의 오리엔테이션이 세로(portrait)인지 가로(landscape)인지에 따라 스타일을 변경할 수 있습니다.

1.3. 미디어 쿼리의 역할

  • 화면 크기에 따라 다른 스타일을 적용합니다.
  • 기기 해상도, 오리엔테이션에 맞춰 스타일을 적용합니다.

2. 미디어 쿼리 적용하기

미디어 쿼리를 사용하여 화면 크기나 장치 특성에 따라 웹 페이지의 스타일을 변경할 수 있습니다. 이를 통해 반응형 웹 디자인을 구현할 수 있습니다.

2.2. 기타 예시

이 예시에서는 3가지 화면 크기에 맞춰 각각 다른 배경색을 적용합니다.

    • 반응형 웹 페이지를 만들 때, @media 규칙을 사용하여 모바일, 태블릿, 데스크탑에서 각각 적합한 스타일을 적용할 수 있습니다.
    • 그리드 레이아웃과 플렉스박스를 활용하여 유연한 레이아웃을 설계할 수 있습니다.

미디어 쿼리는 다양한 화면 크기와 장치에서 최적화된 사용자 경험을 제공하는 데 필수적인 도구입니다.

플렉스 박스 레이아웃 (Flexbox Layout)

플렉스 박스(Flexible Box Layout)는 복잡한 레이아웃을 쉽게 만들 수 있는 CSS3의 레이아웃 모델로, 주로 1차원 레이아웃(가로 또는 세로)에서 요소들을 유동적으로 배치하고 정렬하는 데 사용됩니다. 플렉스 박스를 사용하면 항목들의 크기, 간격, 정렬을 쉽게 조정할 수 있습니다. 기존의 float 이나 position 을 사용한 레이아웃보다 훨씬 더 유연하고 효율적입니다.

1. 플렉스 박스 레이아웃과 기본 속성들

플렉스 박스는 부모 요소(컨테이너)와 자식 요소(항목)로 나뉘며, 부모 요소에 display: flex를 설정하고, 자식 요소에 다양한 정렬 및 크기 조정을 설정할 수 있습니다.

1.1. display: flex

  • 부모 요소에 display: flex를 설정하면, 자식 요소들이 플렉스 항목이 되어 플렉스 컨테이너로 작동합니다.

예시:

  • 이렇게 설정하면, .container 요소의 자식들이 플렉스 항목이 되어 가로 방향으로 나열됩니다.

1.2. 주축과 교차축 (Main Axis and Cross Axis)

  • 주축(Main Axis): 플렉스 항목들이 기본적으로 배치되는 방향입니다. flex-direction에 따라 가로(row) 또는 세로(column)로 배치됩니다.
  • 교차축(Cross Axis): 주축과 수직인 축으로, 항목들이 교차축 방향으로 배치됩니다.

1.3. flex-direction

  • flex-direction 속성은 플렉스 항목들의 배치 방향을 정의합니다.
값 설명
row (기본값) 항목들을 가로 방향으로 배치
column 항목들을 세로 방향으로 배치
row-reverse 항목들을 가로 방향으로 반대 순서로 배치
column-reverse 항목들을 세로 방향으로 반대 순서로 배치

예시:

  • 이 예시에서는 .container 의 자식들이 세로 방향으로 배치됩니다.

1.4. flex-wrap

  • flex-wrap 은 플렉스 항목들이 한 줄에 모두 배치될 수 없을 때, 항목들을 다음 줄로 넘겨줄지 여부를 결정합니다.
값 설명
nowrap (기본값) 모든 항목들이 한 줄에 배치
wrap 항목들이 여러 줄로 배치
wrap-reverse 항목들이 반대 순서로 여러 줄로 배치

예시:

  • 이 예시에서는 항목들이 여러 줄로 배치됩니다.

1.5. justify-content

  • justify-content 는 주축(가로 또는 세로) 방향으로 플렉스 항목들을 정렬하는 속성입니다.
값 설명
flex-start 항목들을 시작 위치에 정렬 (기본값)
flex-end 항목들을 끝 위치에 정렬
center 항목들을 중앙에 정렬
space-between 항목들 간에 공간을 균등하게 분배 (첫 항목과 마지막 항목은 끝에 배치)
space-around 항목들 간에 공간을 균등하게 분배 (항목 양옆에도 공간이 있음)
space-evenly 항목들 간에 균등한 공간을 배분

예시:

  • 이 예시에서는 플렉스 항목들이 수평으로 중앙에 정렬됩니다.

2. 플렉스 박스 항목 배치를 위한 속성들

2.1. align-items

  • align-items 속성은 교차축(세로 또는 가로) 방향으로 플렉스 항목들을 정렬하는 데 사용됩니다.
값 설명
stretch (기본값) 항목들을 교차축 방향으로 늘려서 정렬
flex-start 항목들을 교차축 시작 위치에 정렬
flex-end 항목들을 교차축 끝 위치에 정렬
center 항목들을 교차축 중앙에 정렬
baseline 항목들의 기준선을 맞춰서 정렬

예시:

  • 이 예시에서는 플렉스 항목들이 세로로 중앙에 정렬됩니다.

2.2. align-self

  • align-self 는 개별 플렉스 항목에 대해 교차축 방향으로 정렬을 다르게 설정할 수 있게 해줍니다.
값 설명
auto (기본값) 부모 요소의 align-items 값을 따름
flex-start 항목을 교차축 시작 위치에 정렬
flex-end 항목을 교차축 끝 위치에 정렬
center 항목을 교차축 중앙에 정렬
baseline 항목을 기준선에 맞춰 정렬
stretch 항목을 교차축 방향으로 늘려서 정렬

예시:

  • 이 예시에서는 개별 항목이 교차축 끝 위치에 정렬됩니다.

2.3. flex-grow, flex-shrink, flex-basis

  • flex-grow: 항목이 사용할 수 있는 공간을 얼마나 확장할지 설정합니다.
  • flex-shrink: 항목이 공간이 부족할 때 얼마나 줄어들지 설정합니다.
  • flex-basis: 항목의 기본 크기를 설정합니다.

이 예시에서는 플렉스 컨테이너를 사용하여 3개의 항목을 수평으로 정렬하고, 항목들 사이에 균등한 공간을 배분했습니다.

요약

  1. 플렉스 박스 레이아웃:

    • display: flex 를 사용하여 부모 요소를 플렉스 컨테이너로 설정하고, 자식 요소는 플렉스 항목이 됩니다.
    • flex-direction 을 사용하여 요소들의 배치 방향을 설정합니다. (가로 또는 세로)

  2. 플렉스 박스 항목 배치를 위한 속성들:

    • justify-content: 항목들을 주축 방향으로 정렬합니다.
    • align-items: 항목들을 교차축 방향으로 정렬합니다.
    • flex-grow, flex-shrink, flex-basis 등을 사용하여 플렉스 항목의 크기와 확장/축소 동작을 설정합니다.

플렉스 박스를 활용하면 다양한 레이아웃을 손쉽게 구현할 수 있으며, 반응형 디자인에도 매우 유용합니다.

📦 박스 모델(Box Model)

모든 HTML 요소는 기본적으로 박스 형태로 구성됩니다.

  • margin: 바깥 여백
  • border: 테두리
  • padding: 안쪽 여백
  • content: 내용

🎯 position 속성

값 설명
static 기본 위치
relative 원래 위치 기준으로 이동
absolute 부모 중 position이 지정된 요소 기준
fixed 화면 고정
sticky 스크롤에 따라 고정/해제 전환

🧱 display 속성

  • block: 블록 요소 (div, p, h1 등)
  • inline: 인라인 요소 (span, a)
  • inline-block: 인라인처럼 배치 + 크기 지정 가능
  • flex: 유연한 레이아웃 (부모에 적용)
  • grid: 격자형 배치 (부모에 적용)
